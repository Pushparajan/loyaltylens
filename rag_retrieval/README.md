# rag_retrieval

Offer embedding, vector search, and RAG retrieval layer for LoyaltyLens. Indexes 200 synthetic loyalty offers in both **pgvector** and **Weaviate**, exposes a FastAPI endpoint, and benchmarks LangChain vs LlamaIndex retrievers.

## Purpose

Ground LLM responses in factual, up-to-date loyalty programme content by providing a vector-search retrieval layer. Reduces hallucinations and ensures offer recommendations reference real programme rules.

## Module Map

| File | Responsibility |
| --- | --- |
| `generate_offers.py` | Generate 200 synthetic offers → `data/offers.json` |
| `embeddings.py` | Embed offers with `all-MiniLM-L6-v2`, store in pgvector + Weaviate |
| `langchain_retriever.py` | LangChain `PGVector` retriever with auto-indexing (`lc_offer_embeddings`) and propensity filter |
| `llama_retriever.py` | LlamaIndex `VectorStoreIndex` retriever, same interface |
| `benchmark.py` | Latency (p50/p95) + precision@5 comparison → `benchmark.json` |
| `api.py` | FastAPI: `POST /retrieve`, `GET /offers/stats` |
| `indexer.py` | Chunk, embed, upsert loyalty documents |
| `retriever.py` | Semantic search + MMR re-ranking over loyalty docs |
| `weaviate_client.py` | Weaviate connection and schema management |

## Inputs

- `data/offers.json` — 200 synthetic offers (generated by `generate_offers.py`)
- Customer context string + propensity score (from `propensity_model`)
- Embedding model: `sentence-transformers/all-MiniLM-L6-v2` (384-dim)

## Outputs

- pgvector table: `offer_embeddings(id, title, embedding vector(384))` — direct pipeline writes
- pgvector collection: `lc_offer_embeddings` — LangChain-managed, auto-indexed on first retriever init
- Weaviate class: `Offer` with `id`, `title`, `description`, `category`, `channel`
- `benchmark.json` — latency and precision@5 results
- FastAPI on port `8003`

## Local Setup

### Prerequisites

| Tool | Version | Install |

---

### 1 — Start infrastructure (Docker)

From the **repo root** in PowerShell:

```powershell
# Start pgvector (port 5432), Weaviate (port 8080), Redis (port 6379)
docker compose up -d postgres weaviate redis

# Confirm all three are healthy
docker compose ps
```

Wait until every service shows `healthy`. pgvector runs the `init-pgvector.sql` init script automatically on first start, which enables the `vector` extension.

To stop:

```powershell
docker compose down
```

To wipe volumes and start fresh:

```powershell
docker compose down -v
```

---

### 2 — Create Python environment (uv)

From the **repo root** in PowerShell:

```powershell
# Create a virtual environment in .venv/
uv venv --python 3.11

# Activate it
.\.venv\Scripts\Activate.ps1

# Install all dependencies (production + dev) from the lockfile
uv sync --dev
```

> If you get a script execution policy error, run:
>
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

---

### 3 — Environment variables

Create a `.env` file at the repo root (copy from `.env.example` if present):

```powershell
Copy-Item .env.example .env   # if it exists
```

Minimum required values:

```dotenv
POSTGRES_URL=postgresql://loyaltylens:loyaltylens@localhost:5432/loyaltylens
WEAVIATE_URL=http://localhost:8080
REDIS_URL=redis://localhost:6379
HF_TOKEN=hf_xxxxxxxxxxxxxxxxxxxx   # see instructions below
```

#### Creating a HuggingFace token

1. Go to [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens) and sign in (or create a free account).
2. Click **New token**.
3. Give it a name (e.g. `loyaltylens-dev`), set **Role** to **Read**, then click **Generate a token**.
4. Copy the token — it starts with `hf_` and is shown only once.
5. Paste it as `HF_TOKEN` in your `.env` file.

> The token is **optional** for `all-MiniLM-L6-v2` because that model is public. It is required if you switch to any gated model (e.g. Llama, Mistral).

Load them into the current PowerShell session:

```powershell
Get-Content .env | ForEach-Object {
    if ($_ -match '^\s*([^#][^=]+)=(.*)$') {
        [System.Environment]::SetEnvironmentVariable($matches[1].Trim(), $matches[2].Trim(), 'Process')
    }
}
```

### 4 — Generate offers and build embeddings

```powershell
# Generate data/offers.json (200 synthetic offers)
python rag_retrieval/generate_offers.py

# Embed all offers and store in pgvector + Weaviate
python rag_retrieval/embeddings.py
```

### 5 — Run the API

```powershell
# Windows — use 127.0.0.1 (0.0.0.0 triggers WinError 10013) and python -m uvicorn
python -m uvicorn rag_retrieval.api:app --host 127.0.0.1 --port 8003 --reload --reload-dir rag_retrieval

# macOS / Linux
python -m uvicorn rag_retrieval.api:app --host 0.0.0.0 --port 8003 --reload --reload-dir rag_retrieval
```

Test endpoints:

```powershell
# Retrieve top-5 offers for a customer
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8003/retrieve `
    -ContentType 'application/json' `
    -Body '{"customer_id":"C001","propensity_score":0.65,"context_text":"frequent coffee buyer"}'

# Embedding index stats
Invoke-RestMethod http://127.0.0.1:8003/offers/stats
```

### 6 — Run benchmarks

```powershell
python rag_retrieval/benchmark.py
```

### 7 — Run tests

```powershell
# All RAG tests
python -m pytest tests/test_rag.py -v

# With coverage
python -m pytest tests/test_rag.py -v --cov=rag_retrieval --cov-report=term-missing
```

Tests cover:

- Embedding shape (384-dim assertion)
- pgvector round-trip (insert → query → verify)
- Retrieval returns exactly `k` results
- Propensity filter correctly excludes offers above threshold

### 8 — Full dev container (optional)

The repo ships a multi-stage `docker/Dockerfile` with uv, Python 3.11, Node 18, and Claude Code pre-installed.

```powershell
docker build -f docker/Dockerfile -t loyaltylens-dev .

# Run interactively, mounting source so edits are live

    --env-file .env `
    -v "${PWD}:/app" `
    -p 8003:8003 `
```

Inside the container all setup steps (2–7) work identically.
